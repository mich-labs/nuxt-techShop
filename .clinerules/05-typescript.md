# TypeScript

Избегай использования `any`.

Запрещено:

- `@ts-ignore` без явного согласования;
- `eslint-disable` без комментария с причиной;
- неявные типы там, где явный тип улучшает читаемость или безопасность.

## Использование any

Использование `any` допускается только в инфраструктурном коде, если это обусловлено ограничениями TypeScript или сторонней библиотеки (в т.ч. сгенерированного Strapi-клиента).

К инфраструктурному коду относятся:

- generic helpers;
- utility types;
- адаптеры между DTO-generic'ами Strapi-клиента и внутренними типами, если типизация иначе становится нечитаемой;
- инфраструктурные базовые классы/composables.

Допустимые причины использования `any`:

- ограничение TypeScript;
- ограничение сгенерированного Strapi typed-клиента;
- ограничение другой сторонней библиотеки;
- невозможность выразить тип без существенного усложнения кода.

Во всех таких случаях необходимо оставить короткий комментарий с причиной, например:

```ts
const data = raw as any; // ограничение generic-типа Strapi-клиента при глубокой populate-вложенности
```

Предпочитай:

- строгую типизацию;
- существующие типы проекта (Model-типы слайсов, `types/routeLinks.type.ts` и т.п.);
- вывод типов TypeScript только там, где он очевиден (например `computed`, простые локальные переменные).

Не создавай новые интерфейсы/типы, если уже существует подходящий — особенно для повторяющихся структур вроде ссылок (`AppLinkProps`) и медиа (`{ url, alt }`), см. `04-architecture-layers.md`.

При анализе проекта не считай существующее использование `any` нарушением автоматически. Сначала оцени контекст: если `any` находится в инфраструктурном коде и обусловлен ограничениями TypeScript/Strapi-клиента/сторонней библиотеки — это допустимое исключение, не нарушение. Сообщай о таком коде как об исключении с объяснением, а не молча пропускай и не переписывай без запроса.

## DTO-типы отдельно от Model-типов

DTO-тип (сырой ответ Strapi) объявляется как именованный алиас рядом с Model в `types.ts` слайса, не инлайнится в сигнатуру функции-маппера:

```ts
// правильно
export type BannerDto = SharedBannerGetPayload<{ populate: '*' }>;
export interface BannerModel { /* ... */ }

function mapBanner(dto: BannerDto, mediaHost: string): BannerModel { /* ... */ }
```

DTO-тип не экспортируется через `index.ts` слайса — см. `04-architecture-layers.md`.
